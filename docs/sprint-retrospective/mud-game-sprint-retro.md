# Mud泥巴游戏 Sprint 回顾报告
## 1. 重构过程反思
### 1.1 重构背景
本次冲刺针对泥巴游戏**高耦合遗留核心代码**：`MudGameCore.java`（上帝类，糅合玩家指令解析、角色移动、怪物生成、战斗逻辑、背包操作、场景切换所有功能），采用3种设计模式解耦：
- 命令模式：拆分玩家指令解析（移动/攻击/背包）
- 状态模式：拆分角色状态（空闲/战斗/死亡/回城）
- 工厂模式：拆分怪物生成（普通怪/BOSS怪/精英怪）

### 1.2 重构遇到的问题
1. 遗留代码无注释，玩家指令与战斗逻辑嵌套太深，初期梳理依赖耗时；
2. McCabe复杂度工具初次计算时，包含了工具类方法，数据失真；
3. 重构后角色状态切换出现偶发BUG，需回溯逻辑。

### 1.3 改进措施
1. 重构前先绘制类依赖图，明确`MudGameCore`的耦合点；
2. 固化McCabe计算规则：**仅统计核心业务方法**，忽略工具类/GET/SET方法；
3. 重构一个模块，立即跑通对应游戏场景测试（如移动场景、战斗场景）。
